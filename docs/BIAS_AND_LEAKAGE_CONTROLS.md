# Bias, overfitting and leakage controls

- Retrieval/component development uses frozen dev-300; generator selection uses the first dev-20; E5 uses a separate dev-80 after excluding those 20. EDA and ID freezing may read eval metadata; “locked” means eval labels/output scores are never used to tune and the frozen pipeline executes once. No retuning follows that execution.
- B3/G2/X1/X2 use identical IDs, generator, prompt, output limit and retry policy. Per question, all arms match B3's actual whitespace-word count. X1 matches graph slot count/length with lower-ranked text. X2 uses intact atomic PrimeKG paths sampled away from linked/target nodes, matched on hop count and nearest verbalized length. Separate post-budget validators reject missing/duplicate slots, malformed canonical paths, hop mismatch, forbidden-node overlap or unequal total budget before generation.
- The generator never receives gold answers, gold PMID/snippets, pipeline IDs, experiment metrics or competing answers. It is explicitly told not to prefer graph evidence and not to interpret an association as causality.
- Q1/Q2 runtime logic cannot observe gold misses. Q1 was rejected on dev because harmed-query rate exceeded rescued-query rate. Q2 was not promoted after that gate.
- Correctness/completeness judging receives only question, reference and candidate. Faithfulness judging receives only candidate and cited evidence. Pipeline/retrieval scores, provider IDs and explicit modality cues are masked. Subgroups remain exploratory because they overlap and are not multiplicity-adjusted.
- PrimeKGQA test is accessed only after graph linking/relation canonicalization freeze. Because the RDF compatibility gate failed, SPARQL execution accuracy is not reported as valid; normalized-pattern results remain a component benchmark, not clinical validation.
- Human review uses 100 IDs sampled from locked BioASQ `eval` before locked outputs. Two qualified reviewers see randomized anonymous A/B answers; AI cannot substitute for them.
- Confirmatory claims about the frozen heuristic augmentation require paired statistics and must survive B3, X1 and X2 in ITT plus an eligible complete-X2 sensitivity analysis. API/quota failures remain in the population with conservative scores; they cannot silently change it. A stronger claim about medically relevant graph paths additionally requires human path-usefulness and graph-answerability labels.
- V8 was invalidated from final answer-stage use solely by evidence/control audits before machine judging; no v8 correctness score informed the v9 prompt or retriever. This preserves the dev outcome boundary while allowing a design bug to be fixed transparently.
