# ADR 0003: Stream with Server-Sent Events

- Status: accepted
- Date: 2026-07-12

## Context

After submission, progress, evidence and generated tokens flow from server to browser. The client does not need a bidirectional persistent channel.

## Decision

Submit by HTTP POST and stream events over SSE. Include event IDs, heartbeat comments, reconnect via `Last-Event-ID` and exactly one terminal event.

## Consequences

SSE is simpler than WebSocket and works with native browser support. Cancellation remains an HTTP DELETE. If future interaction becomes truly bidirectional, record a new ADR before changing transports.
