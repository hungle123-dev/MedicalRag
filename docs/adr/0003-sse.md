# ADR 0003: Stream with Server-Sent Events

- Status: accepted
- Date: 2026-07-12

## Context

After submission, persisted job-state snapshots flow from server to browser. The client does not need a bidirectional persistent channel.

## Decision

Submit by HTTP POST and stream `status` snapshots over SSE. Include connection-local event IDs and heartbeat comments, then close after one terminal state. `Last-Event-ID` resume and token streaming are not implemented.

## Consequences

SSE is simpler than WebSocket and works with native browser support. Cancellation remains an HTTP DELETE. If future interaction becomes truly bidirectional, record a new ADR before changing transports.
