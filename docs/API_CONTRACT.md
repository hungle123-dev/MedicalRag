# API contract v1

All product routes use `/api/v1`; `/health` and `/ready` remain compatibility aliases. Generated FastAPI `openapi.json` is authoritative for typed HTTP request/response bodies; this document defines the SSE wire events.

| Method | Route | Behavior |
|---|---|---|
| GET | `/api/v1/health` | Liveness without model loading |
| GET | `/api/v1/ready` | Dependency and B0–G2 availability; 503 while degraded |
| GET | `/api/v1/pipelines` | Allowlisted B0–G2 plus config hash |
| POST | `/api/v1/questions` | Validate and queue one demo request; 202 |
| GET | `/api/v1/questions/{id}` | Persisted queued/running/completed/failed/cancelled state |
| GET | `/api/v1/questions/{id}/events` | SSE state snapshots, monotonic IDs and heartbeat comments |
| DELETE | `/api/v1/questions/{id}` | Cooperative cancellation; artifacts are never deleted |

Submit JSON rejects unknown fields. `question` is trimmed and must contain 3–2,000 characters; `pipeline_id` is exactly B0–B3 or G1–G2; `run_kind` is `demo`.

```json
{
  "question": "Why can propranolol worsen asthma?",
  "pipeline_id": "G2",
  "client_request_id": "optional-client-id",
  "run_kind": "demo"
}
```

The 202 response contains the persisted job (`id`, pipeline, status and timestamps) plus `stream_url`. SSE `status` events contain the persisted job shape without `stream_url`; the terminal completed snapshot includes `result`. SSE sends state snapshots and the terminal answer, not model-token streaming or cross-connection `Last-Event-ID` replay.

Shared result fields are `answer`, `citations`, `evidence`, `details` and `provenance`. Text evidence IDs are stable `PMID:<id>` values. Graph evidence IDs are `primekg:path:<sha256>` over canonical ordered path JSON. Citation validation only accepts IDs in the request evidence registry; invented IDs cause the answer to be suppressed.

Text URLs are generated by the backend from trusted corpus fields and the frontend only links HTTPS `pubmed.ncbi.nlm.nih.gov`. Model text is rendered as text, never HTML. Secrets are environment-only and errors expose stable class codes rather than provider messages or keys.

Cancellation prevents a late result from replacing `cancelled`, although an already-running synchronous retrieval or provider call cannot be forcibly interrupted. Concurrency defaults to two jobs. CORS defaults to the local Vite origins and is configurable through `MEDICAL_RAG_CORS_ORIGINS`.

`/ready` validates the local SQLite connection, loads the configured indexes, checks required PrimeKG tables, and checks that the selected generator is configured. It deliberately does not make a billed provider request; `provider_reachability_checked` is therefore false.
